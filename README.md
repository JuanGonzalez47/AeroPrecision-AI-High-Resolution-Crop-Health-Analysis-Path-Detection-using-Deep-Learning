## Overview
This repository provides a fully autonomous pipeline for precision agriculture, leveraging deep learning (EvolvableDroneNet based on Inception V3) to transform raw high-resolution drone imagery into professional crop health and path maps. The workflow covers everything from raw data processing, curation, model training, evaluation, and high-precision inference.

## Data Directory Structure

Your `data/` folder must be organized as follows:

```
data/
	curated_data/
		crop_healthy/
		crop_stressed/
		path/
		soil/
			# Each contains curated images for that class

	dataset_model/
		test/
			crop_healthy/
			crop_stressed/
			path/
			soil/
		train/
			crop_healthy/
			crop_stressed/
			path/
			soil/
		val/
			crop_healthy/
			crop_stressed/
			path/
			soil/
			# Each split contains images for each class

	prepro_dataset_classifier/
		1_Crop_Healthy/
		2_Crop_Stressed/
		3_Potential_Soil_Path/
			# These are generated by the NDVI preprocessing script. The 'path' folder may need to be created manually.

	raw_data/
		multispectral-images/
			GRE/
			NIR/
			RED/
			REG/
				# Each contains multispectral images
		rgb-images/
			# Contains raw RGB images
```

**Notes:**
- `curated_data/` contains the manually curated images for each class.
- `dataset_model/` contains the train/val/test splits, each with subfolders for the four classes.
- `prepro_dataset_classifier/` is generated by the NDVI preprocessing script; the path folder may need to be created manually.
- `raw_data/` contains the original multispectral and RGB images, organized as shown.

Ensure this structure is followed for the scripts and pipeline to work correctly.

### 1. Raw Dataset
- The original drone imagery dataset can be downloaded from Kaggle:
  [Mixed-Use Agricultural Fields (Kaggle)](https://www.kaggle.com/datasets/marcosgabriel/mixeduse-agricultural-fields?resource=download)

### 2. Curated Data & Model Dataset
- The curated_data and dataset_model folders are not publicly available due to manual curation and balancing.
- To obtain access, please email: **juan.gonzalez57@udea.edu.co**
- Alternatively, you can generate them yourself by following these steps:
  1. **Run the preprocessing script:**
	  - Execute `utils/prepro_agrisense_ndvi_tiling.py` on the raw Kaggle dataset to generate initial tiles and NDVI-based pre-classification.
  2. **Manual curation:**
	  - Review and manually move any misclassified or ambiguous images into a new folder called `curated_data`.
  3. **Split curated data:**
	  - Run `utils/split_data.py` on your `curated_data` folder to create the `dataset_model` directory with balanced train/val/test splits.
	  - Ensure class balance for optimal model performance.

## Project Structure

- `notebooks/pipeline_train_model.ipynb`: The main notebook containing the entire pipeline, including data loading, model definition, training, validation, metrics, plots, and a detailed project conclusion. Follow this notebook for a step-by-step guide from raw data to final results.
- `models/train_model.py`: Standalone script to train the best model (Inception V3 with optimal hyperparameters) using local, relative paths.
- `models/inference.py`: Script for high-resolution inference, generating pixel-voting health/path maps from new drone images.
- `models/INCEPTION-V3/`: Contains the trained model weights (`best_inception_model.pth`, `best_inception_model_script.pth`).
- `utils/prepro_agrisense_ndvi_tiling.py`: Script for NDVI-based tiling and initial dataset preparation.
- `utils/split_data.py`: Script to split curated data into balanced train/val/test sets.
- `docs/04_CNN_Miniproyecto.pdf`: Project statement and requirements document.

## How to Use

1. **Prepare the dataset:**
	- Download the raw dataset from Kaggle.
	- Run the preprocessing and splitting scripts as described above, or request access to curated data.
2. **Follow the pipeline:**
	- Open `notebooks/pipeline_train_model.ipynb` for a complete, reproducible workflow: data loading, augmentation, model selection, training, validation, metrics (accuracy, confusion matrix, classification report), and final conclusions.
3. **Train the model:**
	- Use `models/train_model.py` to train the best model with local paths and optimal hyperparameters.
4. **Inference:**
	- Use `models/inference.py` to generate health/path maps from new high-resolution images.
5. **Model weights:**
	- Pretrained weights are stored in `models/INCEPTION-V3/`.
6. **Project statement:**
	- See `docs/04_CNN_Miniproyecto.pdf` for the original project requirements and objectives.
